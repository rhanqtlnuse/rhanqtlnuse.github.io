1    处理 NULL

经常会遇到处理 NULL 的情况（比如 OUTER JOIN）

IFNULL

    IFNULL(expr, alternative)

如果 expr 不为 NULL，使用 expr 的值；否则，使用 alternative 的值。

5    窗口函数（Window Functions）

MySQL 从 8.0 版本开始支持窗口函数。所谓“窗口”，其实就是分组。

窗口函数对一组查询行执行类似聚合的操作。但是，不同于聚合操作将每个分组“聚合”为单个结果行，窗口函数为分组中的每个行 都生成结果（即结果集的大小不会因窗口操作而变化）。

术语

  • 发生函数求值的行称为当前行
  • 与发生函数求值的当前行相关的查询行构成当前行的窗口

查询中的每个窗口操作都通过包含 OVER 子句来表示，语法如下：

    OVER (
      [PARTITION BY partitioning-criteria
          [ORDER BY ordering-criteria
           [frame-spec]
    )
  • PARTITION BY 子句决定如何分组：如果未指定，它将整个查询行集视为一个分区；否则，根据 PARTITION BY 指定的表达式进行分组，函数会在跨越分区边界时重新初始化；
  • ORDER BY 子句决定分组内 的顺序，注意不要与 SELECT 的 ORDER BY 子句混淆

Window functions are permitted only in the select list and ORDER BY clause.

在 MySQL 中，很多聚集函数也可以作为窗口函数使用。MySQL 也提供了一些只作为窗口函数使用的非聚集函数。

出于复用的目的，可以使用 WINDOW window_name AS (…) 子句定义一个窗口，然后在 OVER 中引用（不是所有数据库都支持）

ROW_NUMBER / RANK / DENSE_RANK

这三个函数都是按分区为结果集中的每一行分配一个排名（rank），不同之处在于：

  • ROW_NUMBER 给相同的值分配不同的排名
    • 有 ROW_NUMBER 时 DISTINCT 不会生效，因为 ROW_NUMBER 在应用 DISTINCT 之前就已经在整个分区上生成唯一值
  • RANK 给相同的值分配相同的排名（即，并列的项排名相同），而且排名不一定连续
  • DENSE_RANK 类似 RANK，但排名是连续的

比如有序列 [a, a, b]，ROW_NUMBER 的结果是 [1, 2, 3]，RANK 的结果是 [1, 1, 3]，DENSE_RANK 的结果是 [1, 1, 2]

记忆

  • ROW_NUMBER 顾名思义 —— 行号，不同的行就应该有不同的行号
  • RANK 类似于体育的排名，会出现并列的情况
  • DENSE_RANK 看名字也能了解大概 —— DENSE 有“稠密”的意思，因此它会给出连续的排名

OVER 子句的完整文法

    over-clause ::= 
      OVER ([partition-clause]
            [order-clause]
            [frame-clause])

    partition-spec ::= 
      expr
      | expr, partition-spec

    order-spec ::= 
      expr [ASC|DESC]
      | expr [ASC|DESC], order-spec

    frame-spec ::= frame-units frame_extent

    frame-units ::= ROWS | RANGE

    frame_extent ::= frame_start | frame_between

    frame_between :: = BETWEEN frame_start AND frame_end

    frame_start, frame_end ::=
      CURRENT ROW
      | UNBOUNDED PRECEDING
      | UNBOUNDED FOLLOWING
      | expr PRECEDING
      | expr FOLLOWING
