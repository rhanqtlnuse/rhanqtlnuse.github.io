Shell 脚本编程

目前为止，我们已经看到如何在 Shell 中执行命令以及用管道将若干个命令连接起来。然而，在许多场景下你可能会想要执行一系列的命令，并在其中使用条件和循环这样的控制流表达式。

就复杂度而言，Shell 脚本是 Shell 命令的下一步。大多数 Shell 有它们自己的脚本编程语言（有变量和控制流）和语法（syntax）。Shell 脚本编程与其他的脚本编程语言的区别在于，Shell 脚本编程为执行 Shell 相关的任务进行了优化。因此，创建命令流水线、将结果保存到文件或者是从标准输入中读取在 Shell 脚本编程中是基本（primitive）操作，这样就使 Shell 脚本比通用（general purpose）的脚本编程语言更易用。在本节中，我们会集中于最常见的 Bash 脚本编程。

在 Bash 中，给变量赋值的语法为 foo=bar，读取变量值的语法为 。注意：foo = bar 不会正常运行，因为它会被解释为以 = 和 bar 为参数调用 foo。通常来说，Shell 脚本中的空格会用于分隔参数，初学者通常会对此感到困惑，因此记得经常检查这一点。

Bash 中的字符串可以由 ' 和 " 两个定界符定义，但是二者并不等价。由 ' 定界的字符串是字面量，不会进行变量值替换，而由 " 定界的字符串会进行变量替换。

如大多数编程语言一样，Bash 支持 if case while for 控制流。同样，Bash 也有接收参数（argument）并用这些参数进行操作的函数。下面的例子展示了创建一个目录并 cd 进入其中的函数的例子。

这里，$1 是脚本 / 函数的第一个参数。不像其他脚本编程语言，Bash 使用许多特殊的变量来引用参数、错误码和其他相关的变量。下面列出了一部分，更完整的列表可以从这里找到。

  • $0 - 脚本的名字
  • $1 到 $9 - 脚本的参数，$1 表示第一个参数，以此类推
  • $@ - 所有的参数
  • $# - 参数的个数 1
  • $? - 前一个命令的返回代码（return code）
  • $$$$ - 当前脚本的进程标识号
  • !! - 完整的前一个命令，包括参数。常见的用途是执行一个仅由于权限问题而失败的命令，你可以便捷地加上 sudo 重新执行它 sudo !!
  • $_ - 前一个命令的最后一个参数。如果你在使用交互式 Shell，你可以通过输入 Esc 加上 . 快速地得到这个值

命令通常会将输出写至 STDOUT、将错误写至 STDERR，并以对 Bash 更加友好的方式返回一个返回码来报告错误。返回码或者退出状态（exit status）是脚本 / 命令指示执行如何进行的方式。值 0 通常表示一切 OK，其他的值表示有错误发生。

另一种常见的模式是将命令的输出赋值给一个变量。这可以通过过命令替换（command substitution）实现。对于 ，Bash 会执行 CMD 并且得到命令的输出，将其原地替换。例如，如果你写了 for file in $(ls)，Shell 会首先调用 ls，然后遍历这些值。一个相似但是知道的人比较少的特性是进程替换（process substitution），<(CMD) 会执行 CMD，然后将输出写入一个临时文件，并且将 <() 替换为这个文件的名字。当命令希望通过文件而不是标准输入传递值时，这个特性很有用。例如，diff <(ls foo) <(ls bar) 会显示 foo 和 bar 两个目录中文件的不同。

运行脚本时，你可能常常会想要提供一些相似的参数，Bash 为此提供了简化的方法：通过执行文件名展开（filename expansion）来展开表达式。这些技巧通常被称为 Shell globbing。

  • 通配符（wildcard）- 你可以使用 ? 匹配一个任意字符，用 * 匹配任意数量的任意字符。例如，给定文件 foo foo1 foo2 foo10 bar，命令 rm foo? 会删除文件 foo1 和 foo2，而命令 rm foo* 会删除 bar 以外的所有文件

  • 花括号（curly braces）- 如果你想要执行一系列的命令，并且命令中有相同的子串，那么可以使用花括号来自动展开它们。当你想要移动或是转换文件时，这个功能很方便：

下面是 shell 函数和脚本之间的一些不同点，你应当牢记它们：

  • 函数和 shell 必须是同一种语言，而脚本可以用任何语言编写。这就是为什么脚本中包含 shebang 很重要。
  • 函数只在它们的定义被读取时加载一次，而脚本在每次执行时都会加载。这一点让函数的加载会稍稍快一点，但是如果你更改了函数的定义，就必须重新加载。
  • 函数在当前的 shell 环境中执行，而脚本在它们自己的进程中执行。因此，函数能够修改环境变量（比如，修改当前目录），而脚本不能。脚本的环境变量是那些用 export 导出的，并且是值传递。
  • 跟其他的编程语言一样，函数是一种强大的构造，能够实现 shell 代码的模块化、代码重用和清晰。通常 shell 脚本会包含自己的函数定义。

Shell 工具

查找如何使用命令

现在，你可能想知道如何找到命令在别名部分（aliasing section）的标志（flag）（比如 ls -l mv -i mkdir -p）。更一般地，你如何了解任何一个命令的功能及其选项？你应该会去 google，但是 UNIX 毕竟诞生在 StackOverflow 之前，所以它们有获取这些信息的内置方法。

如我们在 shell 这一讲里看到的，首先应该想到的方法是用 -h 或 --help 标志来调用命令。要想获取更详细的信息，可以使用 man 命令（manual 的缩写），它会给你显示指定命令的手册页（manual page，在 UNIX 中叫做 manpage）。例如，man rm 会输出 rm 的行为以及它所允许的不同的选项（包括前面提到的 -i）。事实上，我目前为止给出的命令的链接是对应命令的 Linux 手册页的在线版本。即使是非原生的命令也会有手册页，如果开发者有写并且将其作为安装过程的一部分的话。对于交互式工具（比如基于 ncurse），通常可以在程序中使用 :help 或 ? 来获取命令的帮助。

有时，手册页可能描述得过于详细，并且很难解释对于常见用例中使用的标志/语法。TLDR 页面是一个很好的补充解决方案，其重点是给出命令的示例使用案例，以便你可以快速确定要使用的选项。例如，我发现自己参考 tar 和 ffmpeg 的 tldr 页面的频率要高于手册页。

每个程序员都会面临的最常见的重复性任务之一就是查找文件或目录。所有 UNIX-like 的系统都带有 find，这是一个查找文件的好工具。find 将递归搜索符合某些条件的文件。一些例子：

除了列出文件之外，find 还可以对与你的查询匹配的文件执行操作。此属性对于简化可能非常单调的任务非常有用。

尽管 find 无处不在，但记住它的语法有时仍然很棘手。例如，要找到与某个模式 PATTERN 匹配的文件，你必须执行 find -name '*PATTERN*'（如果希望模式匹配不区分大小写，则执行 -iname）。你可以为这些场景构建别名，但是作为 shell 哲学的一部分，使用替代方案是比较好的。请记住，shell 的最佳特性之一就是你只是在调用程序，因此你可以找到（甚至自己编写）某些程序的替代品。例如，fd 是 find 的一种简单、快速且用户友好的替代方案。它提供了一些不错的默认值，例如彩色输出、默认正则表达式匹配、Unicode 支持，而且我认为它的语法更直观。查找模式 PATTERN 的语法为 fd PATTERN。

大多数人都认为 find 和 fd 很好，但是有些人可能想知道每次查找文件的效率与编译某种索引或数据库以快速搜索相比是否有效。这就是 locate 的目的。locate 依赖于使用 updatedb 更新的数据库。在大多数系统中，updatedb 每天通过 cron 更新。因此，两者之间的权衡是速度与新鲜度。此外，find 和类似工具还可以使用诸如文件大小、修改时间或文件权限之类的属性来查找文件，而 locate 仅能使用名称。在这里可以找到更深入的比较。

查找代码

查找文件很有用，但通常情况下，你会关注文件中的内容。一种常见情况是希望搜索包含某种模式的所有文件，以及这些模式在这些文件中的位置。为了实现这一点，大多数 UNIX-like 的系统都提供了 grep，这是一个用于匹配输入文本中的模式的通用工具。它是一个非常有用的 shell 工具，我们将在 data wrangling 这一讲中更详细地介绍它。

grep 有很多标志，使其成为非常通用的工具。我经常使用的一些命令是：-C 用于获取匹配行的上下文（context），以及 -v 用于反转（invert）匹配项，即打印所有与模式不匹配的行。例如，grep -C 5 将打印匹配行的前后 5 行。当涉及到快速解析许多文件时，使用 -R，它将递归地进入目录并查找匹配字符串的文本文件。

但是 grep -R 可以通过多种方式进行改进，例如忽略 .git 文件夹、使用多 CPU 支持等等。因此，不乏开发出的替代品，包括 ack，ag 和 rg。它们都很棒，但是基本上满足的是相同的需求。就目前而言，因为 ripgrep（rg）的快速和直观，我一直在使用它。一些例子：

查找 Shell 命令

到目前为止，我们已经看到了如何查找文件和代码，但是随着你在 Shell 中花费越来越多的时间，你可能希望查找你在某个时刻键入的特定命令。首先要知道的是，按 ↑ 键能够返回上一条命令，如果继续按该键，你就能龟速浏览 Shell 历史记录。

history 命令会把你的 Shell 历史记录打印到标准输出中。如果要在结果中搜索，可以将输出通过管道传递到 grep 并搜索模式。history | grep find 将打印带有子字符串“find”的命令。

在大多数 Shell 中，你可以使用 Ctrl + R 在历史记录中进行后向搜索。按下 Ctrl + R 后，你可以输入想要在历史记录中匹配的子字符串。继续按下该组合键，你将循环浏览历史记录中的匹配。在 zsh 中，这也可以通过 ↑ / ↓ 键实现。fzf 绑定是 Ctrl + R 的一个不错的补充。fzf 是通用模糊（fuzzy）查找器，可以与许多命令一起使用。这里用于模糊匹配你的历史记录，并以方便且直观的方式呈现结果。

另一个我非常喜欢的与历史相关的技巧是基于历史的自动建议（history-based auto-suggestions）。最初由 fish shell 引入，此功能能够根据你最近输入的与当前的 shell 命令有公共前缀的命令自动补全当前的命令。zsh 支持这种功能，这是一个很棒的技巧。

最后，要记住的一点是，有前导空格的命令不会加入到历史记录中，这一点在你输入带有密码或其他敏感信息的命令时很有用。如果你忘记了添加前导空格，可以随时通过编辑 .bash_history 或 .zhistory 手动删除条目。

目录导航

到目前为止，我们一直假定你已经在执行这些操作所需的位置，但是如何快速浏览目录呢？你可以通过许多简单的方法来执行此操作，例如编写 shell 别名（alias）、用 ln -s 创建符号链接，不过实际上到目前为止，开发人员已经找到了非常聪明和精巧的解决方案。

与本课程的主题一样，你通常希望针对常见情况进行优化。可以使用 fasd 之类的工具来查找频繁和 / 或最新的文件和目录。fasd 可以根据频率（frecency）（即频率和新近度）对文件和目录进行排名。最直接的用法是 autojump，它添加了 z 命令，你可以使用该命令使用最新目录的子字符串快速进行 cd。例如，如果你经常进入 /home/user/files/cool_project，你可以用 z cool 跳到那里。

也有更复杂的支持快速获得目录结构概览的工具，比如 tree、broot 以及成熟的文件管理器，比如 nnn 或 ranger。
