2.1    EXISTS + 关联子查询

关联子查询意味着，对于筛选出的每一条记录，都要运行一次子查询。因此，当 EXISTS 前面有高效的过滤条件时，适合使用，但是当子查询是唯一条件时，还是使用 IN 比较好。

注意：不要用 COUNT(*) 做存在性检查，因为优化器实际上会找到所有符合条件的记录，而 EXISTS 会在找到第一条符合条件的记录时停止。
