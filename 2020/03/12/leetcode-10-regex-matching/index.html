LeetCode 10：正则表达式匹配

发表于 2020-03-12
困难

题目见 LeetCode

回溯题。

思路其实并不难。一共有三种字符，但实际上我们可以将它们分为两类：

  • 用于进行字面匹配的字符：字母和 .
  • 量词：* 和缺省

所以一共有四种组合：

  • 字母 + 缺省
  • 字母 + *
  • . + 缺省
  • . + *

关键就是对这四种组合的处理，再加上判断字母和字母匹配的逻辑就好了。

有这样几种情况：

  • s[i] == p[j]（p[j] 是字母）
    • 如果 p[j] 后面带 *，那么我们就要分别考察
      • 与 p[j]* 匹配 0 次（即越过）
      • 与 p[j]* 匹配多次
    • 如果 p[j] 后面不带 *，那就好办了，i 和 j 都可以继续向前
  • s[i] != p[j]（p[j] 是字母）
    • 这种情况下唯一的可能就是能够跳过 p[j]，即 p[j] 带 ，如果 p[j] 不带 `\`，说明无法匹配
  • p[j] == ‘.’
    • 由于 ‘.’ 能够匹配任意字符，所以实际上这种情况下的逻辑与 s[i] == p[j] 的处理相同

递归基：

  • 如果 p 是空串，意味着只能够匹配 ""，s 是否是空串就能够决定
  • 如果 s 是空串，就要注意了，p 不一定为空，只要能够匹配空串即可，具体如下：
    • 如果 p 能够匹配空串，其中的每一个部分一定都是带 * 的，因此其长度必然是偶数；
    • 接着考察每一个偶数位置（如果从 0 开始就是索引为奇数的位置）是否是 *

另外，还有一个问题：当遇到 <token>* 时，可能每次都会产生两个分支，因此，我们要看是否能够对这里进行优化。

事实上是可以的，因为可能会出现 "a*a*a*a*a*a*a*a*a*" 这样的数据，而显然后面的 a* 都是没有意义的，可以将它们合并。
