数据库 | SELECT 语句的执行顺序

发表于 2020-03-25 分类于 数据库 (Database) ， SQL
介绍 SQL 中 SELECT 语句各部分的执行顺序

其中，SELECT 语句的执行顺序：

  1. FROM 和 JOIN
    • 包括其中的子查询
    • 可能会产生临时表
  2. WHERE
    • 只能访问 FROM 子句中所有的表所包含的列
    • 在大多数数据库中，此时还不能使用在 SELECT 子句中定义的别名，因为这些别名可能包含依赖于未执行的部分的表达式
  3. GROUP BY
    • 分组的结果的行数与指定分组列中不同值的个数相同。这其实隐含地表明，应当只在需要进行聚合查询时才使用 GROUP BY，否则就会丢失信息
  4. HAVING
    • 与 WHERE 子句类似，大多数数据库也不允许在 HAVING 子句中使用 SELECT 中的别名
  5. SELECT
  6. DISTINCT
  7. ORDER BY
    • 因为 SELECT 已经执行过，所以现在可以使用 SELECT 子句中定义的别名了
  8. LIMIT 和 OFFSET
