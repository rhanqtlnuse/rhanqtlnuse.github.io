我们所求的是最大的加权和，那么自然，如果较大的（正）值有较大的权重，最终的加权和就会比较大，为了方便地得到较大的值，可以先给整个数组从大到小排个序。

我们希望较大的值的权重越大越好，而权重跟序列的长度正相关（这里表述不太严谨），但是因为数组中会出现负值，所以并不是序列并不是越长越好，比如题中的示例 1：

    satisfaction = [-1, -8, 0, 5, -9] -> [5, 0, -1, -8, -9]

能得到最大的加权和 14 的序列长度为 3（5, 0, -1），如果序列再长一点，变成 5, 0, -1, -8，加权和就变为 10，因此序列并不是越长越好，那么如何找到适当的序列长度呢？

我们可以先从只选一个的情况入手。这时，我们只需看看是否有最大值 $s_1 > 0$，如果是的话，那么我们的最大加权和就是 $s_1$，相反，如果最大值小于或等于零，那么什么都不选就能够得到最大加权和 0。
