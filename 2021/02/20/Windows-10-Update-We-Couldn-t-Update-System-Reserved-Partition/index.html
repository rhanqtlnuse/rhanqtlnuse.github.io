Windows 10 更新报错：我们无法更新系统保留分区

发表于 2021-02-20 更新于 2021-04-15
本文探究 Windows 10 更新过程中“无法更新系统保留分区”问题的解决方案

在使用 Windows 10 易升工具更新系统时，最后总会报错：我们无法更新系统保留的分区，错误代码是 0x1900201。

使用中文关键字搜索，基本上都应该能看到这两篇博客：【Win10】我们无法更新系统保留的分区 - 博客园 和 【DIY小记】踏平Win10升级之坑——我们无法更新系统保留的分区 - CSDN。两篇博客都说错误的原因是系统保留分区的空间不足，提到的步骤也差不多：（1）备份引导分区；（2）在 WinPE 中删除引导分区和系统盘上的 MSR 分区；（3）创建新的更大的引导分区和 MSR 分区；（4）从之前的备份中恢复引导分区数据。

因为安装了双系统，我担心直接操作引导分区会出问题，所以就用 WinPE 扩大了 MSR 分区的容量（当然在这之前还是备份了引导分区），不过还是报错。

再次搜索，微软官方论坛中也说是因为系统保留分区空间不足，用中文搜索出来的其他解决方案基本差不多，所以尝试用英文搜索，搜到了微软官方的一篇文档，用这里面的方法，最后发现所谓的“系统保留的分区”并不是 MSR 分区，而是 SYSTEM 分区（不得不说 MicroSoft Reserved partition 这个名字有一定的误导性）。

调整 SYSTEM 分区的大小后，就能更新成功了。
