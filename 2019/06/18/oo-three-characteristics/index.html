面向对象程序设计：浅谈三个基本特性

发表于 2019-06-18 更新于 2019-06-26

相信很多人在面试的时候都会被问到这样一个问题：面向对象编程有哪些基本特性？说说你的理解。本文就记录一下个人对于这三个特性的理解。

封装

我们先从封装说起。其实，“封装”这个概念并不是面向对象编程特有的，它存在于计算机科学的各处。最直接的，支持面向对象编程范式的语言，比如 C++、Java、Python 都提供了“类”的机制（Python 还有“模块”）；更进一步，其实像 C 这样的过程式的语言也提供了封装的机制 —— 函数（不同于 C++，C 中的结构体只是一种“组合”的方法）；再看远一点，我们学数据结构，都会提到“ADT”，这也是一种封装的体现；还有，我们的计算机系统就是层层建立起来的，每一层都在封装之后对外部提供了接口。

封装带来的最大好处是模块化和信息隐藏（安全性）。模块化其实就是抽象，它能够：

  • 使得模块容易“插拔”。比如，我们可以在用户完全没有感觉的情况下，用有相同接口的新模块替换旧模块；
  • 降低心智负担。在观察、理解一个系统时，我们可以每次只关注这个系统的一个部分/层次/组件，将主要的精力放在理解当前的部分，而不是使用大量的精力在脑中维护各种复杂的调用、跳转关系。

透明

计算机科学中的“透明”，与我们生活中所说的“透明”是截然相反的意思：生活中常说的“透明”，指的是我们完全（能够）了解一个系统的内部构造；而计算机科学中的“透明”，则指的是我们可以完全将一个系统作为一个“黑盒子”看待，完全不需要了解其内部构造也可以毫无障碍地使用。

继承

继承是为了复用。衍生类能够直接使用父类的数据、方法而不需要重新实现。当然，被继承的类也有一些控制继承的方法，比如 Java 中的 final、public、protected 和 private。
