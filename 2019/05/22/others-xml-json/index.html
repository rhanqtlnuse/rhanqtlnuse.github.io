XML vs. JSON

发表于 2019-05-22 更新于 2020-03-25
本文介绍两种常见的数据格式 —— XML 和 JSON，并进行对比

XML

XML 全称为 eXtensible Markup Language，中文为 “可扩展标记语言”。

XML 由 6 个部分组成：

  • 处理指令
  • 根元素
  • 子元素
  • 属性
  • 内容
  • 注释

JSON

JSON 全称为 JavaScript Object Notation，中文为 “JavaScript 对象表示”。相比于 XML，JSON 最大的特点就是简单、容易编写。同时，从名字就可以看得出来，JSON 能够无缝融入 JavaScript 中（Python 也可以），这也是一个极大的优势。

JSON 是一种 key-value 格式的文本，由 6 种类型组成：

  • 对象 (object)
  • 数组 (array)
  • 字符串 (string)
  • 数值 (number)
  • 布尔值：true 或 false
  • null

对比

  • JSON 通常情况下会比 XML 简单，冗余信息少，能解决大多数的需求
  • JSON 能够直接在 JavaScript 中使用，XML 必须解析成 DOM
  • JSON 会丢失一些信息，比如 XML 中 <book> 标签表示这是一个 Book 对象，如果想要在 JSON 中加入这个信息，就只能增加一行 type: "Book"
  • 由于 XML 有 XSD，XSLT 等工具，有些应用场景 XML 比 JSON 更好用（比如将数据文件转换为 HTML）
