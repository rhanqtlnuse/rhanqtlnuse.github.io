C++ 学习笔记 - 基础篇：类的构造和析构

发表于 2019-01-11 更新于 2019-01-17 分类于 Programming Languages ， C++

概述

在介绍 C++ 的类构造和析构之前，先介绍一个重要的原则：RAII。

RAII 源于 C++，是 Bjarne Stroustrup 和 Andrew Koenig) 在设计 C++ 异常时，为解决资源管理的异常安全性而使用的方法，后来 Bjarne 称之为 RAII。

RAII 要求，资源的有效期与持有资源的对象的生命期严格绑定，即由对象的构造函数完成资源的分配（获取），同时由析构函数完成资源的释放。在这种要求下，只要对象能正确地析构，就不会出现资源泄露的问题。$^{[1]}$

在 C++ 中，类的构造函数有以下几种：

  • 转换构造函数

移动构造函数 $^{[6]}$

移动语义是 C++ 11 引入的新特性。

cppreference.com 对移动构造函数的定义如下：类 T 的移动构造函数是非模板构造函数，其首参数是 T&&、const T&&、volatile T&&、const volatile T&&，无其他参数或者剩余参数均有默认值。

语法如下：

  1. class_name(class_name &&);：典型声明
  2. class_name(class_name &&) = default;：强制编译器生成移动构造函数
  3. class_name(class_name &&) = delete;：避免隐式移动构造函数

典型的移动构造函数“窃取”参数曾所保有的资源（例如指向动态分配对象的指针、文件描述符、 TCP 接头、 I/O 流、运行的线程等），而非复制它们，并令参数在某个合法但不确定的状态。

什么是右值？

在继续学习移动构造函数之前，先学习一个概念：右值。

右值表达式包括纯右值、亡值。

纯右值

下列表达式是纯右值表达式：

  • （除了字符串字面量之外的）字面量，例如 42 、 true 或 nullptr；

    • 字符串字面量不是纯右值表达式的原因简略地说就是，像 42、true 这些字面量因为其对应的类型（int、bool）有固定的长度，所以可以很容易地直接写在生成的机器码中（比如 movl %eax, $42），而不同的字符串的长度不总是相同，所以一般情况下是将字符串字面量作为内存中的数据对象，而“数据对象”这一概念就是左值。$^{[4]}$

  • 函数调用或重载运算符表达式，其返回类型是非引用，例如 str.substr(1, 2)、str1 + str2 或 it++；

  • 与运算相关的表达式（个人认为这一部分可以套用上一条，当作是返回类型为非引用的函数调用）

    • a++ 与 a--，内建的后置自增与后置自减表达式；

      • 这一条要特别注意！对应地，++a 和 --a 是左值

      • 原因比较容易想到：因为返回的是 a 之前的值，以如下类定义为例：

    • a + b 、 a % b 、 a & b 、 a << b ，以及其他所有内建的算术表达式；

    • a && b、a || b、!a，内建的逻辑表达式；

    • a < b、a == b、a >= b 以及其他所有内建的比较表达式；

  • 成员访问运算符 $^{[3]}$

    • &a，内建的取地址表达式；

    • 内建的成员访问运算符

      • a.m，对象成员表达式，其中 m 是成员枚举符或非静态成员函数（C++11 前，其中 a 为右值且 m 为非引用类型的非静态数据成员）；
      • p->m，内建的指针成员表达式，其中 m 为成员枚举符或非静态成员函数；

    • 内建的成员指针访问运算符

      • a.*mp，对象的成员指针表达式，其中 mp 是指向成员函数的指针（C++11 前，其中 a 为右值且 mp 为指向数据成员的指针）；

      • p->*mp，内建的指针的成员指针表达式，其中 mp 是指向成员函数的指针；

  • a, b，内建的逗号表达式，其中 b 是右值；

  • a ? b : c，某些 b 和 c 的三元条件表达式；

    • 如果 b 和 c 都是左值呢？
    • 如果 b 和 c 一个是左值一个是右值呢？

  • 转换为非引用类型的转型表达式，例如 static_cast<double>(x)、std::string{} 或 (int)42；

  • this 指针；

    • 原因：this 指针并不能像其他指针那样修改，并且实际上并未存储在某个位置，它的值是由编译器基于上下文确定的。$^{[5]}$对 this 取地址会出错：

  • 枚举项；

  • Lambda 表达式，例如 [](int x){ return x * x; }；(C++11 起)

性质
  • 右值共有的性质
  • 纯右值不能多态：它所标识的对象的动态类型始终为该表达式的类型。
  • 非类非数组的纯右值不能有 cv 限定。（注意：函数调用或转型表达式可能生成非类的 cv 限定类型纯右值，但 cv 限定符被立即剥除。）
  • 纯右值不能具有不完整类型。（除了类型 void（见下文），或在 decltype 指定符中使用之外）

亡值

下列表达式是亡值表达式：

  • 函数调用或重载运算符表达式，其返回类型为右值引用，例如 std::move(x)；
  • a[n]，内建的下标表达式，其操作数之一是数组右值；
  • 成员访问运算符
    • a.m，对象成员表达式，其中 a 是右值且 m 是非引用类型的非静态数据成员；
      • C++ 11 前，这是“右值”
    • a.*mp，对象的成员指针表达式，其中 a 为右值且 mp 为指向数据成员的指针；
      • C++ 11 前，这是“右值”
  • a ? b : c，某些 b 和 c 的三元条件表达式；
  • 转换为对象的右值引用类型的转型表达式，例如 static_cast<char&&>(x)；
  • 任何指代临时对象的表达式，在临时量实质化后。（C++17 起）
性质
  • 右值共有的性质。
  • 泛左值共有的性质。

特别是，同所有的右值类似，亡值可以绑定到右值引用上，而且同所有的泛左值类似，亡值可以是多态的，而且非类的亡值可以有 cv 限定。

右值

右值表达式包括纯右值、亡值。

性质：

  • 右值不能被取地址： &int() 、&i++、&42 及 &std::move(x) 是非法的；
  • 右值不能用作内建赋值运算符及内建复合赋值运算符的左操作数；
  • 右值可以用于初始化 const 左值引用，这种情况下该右值所标识的对象的生存期被延长到该引用的作用域结尾；
  • C++ 11 的扩充
    • 右值可以用于初始化右值引用，这种情况下该右值所标识的对象的生存期被延长到该引用的作用域结尾；
    • 当被用作函数实参且该函数有两种重载可用，其中之一接受右值引用的形参而另一个接受 const 的左值引用的形参时，右值将被绑定到右值引用的重载之上（从而，当复制与移动构造函数均可用时，以右值实参将调用其移动构造函数，复制和移动赋值运算符与此类似）。

隐式声明的移动构造函数

若不对类类型（struct 、class 或 union ）提供用户定义的移动构造函数，且下列所有为真：

  • 无用户声明的复制构造函数；
  • 无用户声明的复制赋值运算符；
  • 无用户声明的移动赋值运算符；
  • 无用户声明的析构函数；
  • 隐式声明的移动构造函数不因下列任何条件成立而定义为被删除（C++14 前）
    • T 有不能移动（有被删除、不可访问或有歧义的移动构造函数）的非静态数据成员；
    • T 有不能移动（有被删除、不可访问或有歧义的移动构造函数）的直接或虚基类；
    • T 有直接或虚基类，带被删除或不可访问的析构函数；
    • T 是类联合类，且拥有带非平凡移动构造函数的变体成员；
    • T 有数据成员或直接或虚基类，它不可平凡复制且无移动构造函数。(C++14 前)
    • 重载决议忽略被删除的隐式声明的移动构造函数（否则它会阻止从右值复制构造）（C++14 起）

则编译器将声明移动构造函数为其类的非 explicit 的 inline public 成员，签名为 T::T(T&&) 。

类能拥有多个移动构造函数，例如 T::T(const T&&) 和 T::T(T&&) 。若存在用户定义的移动构造函数，则用户仍可用关键词 default 强制编译器生成隐式声明的移动构造函数。

移动构造函数的调用时机

从同类型的右值（亡值或纯右值）（C++17 前）（C++ 17 起为“亡值”）初始化（以直接初始化或复制初始化）对象时，调用移动构造函数，情况包括：

  • 初始化： T a = std::move(b); 或 T a(std::move(b)); ，其中 b 有类型 T ；
  • 函数参数传递： f(std::move(a)); ，其中 a 有类型 T 而 f 为 Ret f(T t)；
  • 函数返回：在如 T f() 的函数中的 return a; ，其中 a 有类型 T ，它有移动构造函数。

初始化器为纯右值时，通常会优化掉 (C++17 前)决不进行 (C++17 起)移动构造函数的调用，见复制消除。

初始化列表 $^{[10]}$

上面两种写法结果是相同的，但是过程（或者说意义）不同：使用初始化列表是显式初始化类的成员，而在构造函数体中是对类的成员赋值。

对内置类型的成员（包括指针），初始化和赋值没有什么大的区别，像上面的任一个构造函数都可以。对非内置类型成员变量，为了避免两次构造，推荐使用初始化列表。进入构造函数体之前，成员必须都进行过初始化，可以是显式的，可以是隐式的，但有的时候必须用带有初始化列表的构造函数：

  • 成员类型是没有默认构造函数的类。若没有提供显式初始化，则编译器隐式使用成员类型的默认构造函数；若类没有默认构造函数，则编译器尝试使用默认构造函数将会失败。
  • const 成员或引用类型的成员。因为 const 对象或引用类型只能初始化，不能对他们赋值。

结果 & 性能：

  • 内置数据类型，复合类型（指针，引用）：在成员初始化列表和构造函数体内进行，在性能和结果上都是一样的
  • 用户定义类型（类类型）：结果上相同，但是性能上存在很大的差别。因为类类型的数据成员对象在进入函数体前已经构造完成，也就是说在成员初始化列表处进行构造对象的工作，调用构造函数，在进入函数体之后，进行的是对已经构造好的类对象的赋值，又调用个拷贝赋值操作符才能完成（如果并未提供，则使用编译器提供的默认按成员赋值行为）

不过，这里的 RAII 并不是十分严格：对象（所占的内存也是资源）的创建（资源获取）是在构造函数之外进行的。广义上，我们也把它划归 RAII 范畴。但是，Matthew Wilson 在《Imperfect C++》一书中，将其独立出来，称其为 RRID（Resource Release Is Destruction，资源释放即析构）。RRID 的实施需要在类的开发者和使用者之间建立契约，采用相同的方法获取和释放资源。比如，如果在 shared_ptr 构造时使用 malloc()，便会出现问题，因为 shared_ptr 是通过 delete 释放资源的。

域守卫技术

当一个函数需要通过多个局部变量来管理资源时，RAII 就显得非常好用。因为只有被构造成功（构造函数没有抛出异常）的对象才会在返回时调用析构函数，同时析构函数的调用顺序恰好是它们构造顺序的反序，这样既可以保证多个资源（对象）的正确释放，又能满足多个资源之间的依赖关系。

RAII 的“资源”，不仅仅局限于动态获取（new / delete）的资源，还可以推而广之，用于管理广义的“资源”。假设我们有这样一个多线程的程序，其临界资源每次只能由一个线程来读/写，一种简单的做法就是使用一个标志位，表示临界资源是否被占用：
